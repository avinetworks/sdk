GslbSite Up

address (optional)
String IP Address or a DNS resolvable, fully qualified domain name of the Site Controller Cluster.
cluster_uuid
String UUID of the 'Cluster' object of the Controller Cluster in this site.
dns_vs_uuids (optional)
array[String] The DNS VSes on which the GslbServices shall be placed. The site has to be an ACTIVE member. This field is deprecated in 17.2.3 and replaced by 'dns_vses' field. . Field deprecated in 17.2.3.
dns_vses (optional)
array[GslbSiteDnsVs] This field identifies the DNS VS and the subdomains it hosts for Gslb services. . Field introduced in 17.2.3.
enabled (optional)
Boolean Enable or disable the Site. This is useful in maintenance scenarios such as upgrade and routine maintenance. A disabled site's configuration shall be retained but it will not get any new configuration updates. It shall not participate in Health-Status monitoring. VIPs of the Virtual Services on the disabled site shall not be sent in DNS response. When a site transitions from disabled to enabled, it is treated similar to the addition of a new site.
hm_proxies (optional)
array[GslbHealthMonitorProxy] User can designate certain Avi sites to run health monitor probes for VIPs/VS(es) for this site. This is useful in network deployments where the VIPs/VS(es) are reachable only from certain sites. A typical scenario is a firewall between two GSLB sites. User may want to run health monitor probes from sites on either side of the firewall so that each designated site can derive a datapath view of the reachable members. If the health monitor proxies are not configured, then the default behavior is to run health monitor probes from all the active sites. Field introduced in 17.1.1.
ip_addresses (optional)
